led-color vs state

The led-color is a different story because the led color is in relation to the state of the button...You see..the button not only has 'on' and 'off' states...they also have 'fail' and 'standby' states.
This is born of necessity, because the -elements are 'Gun aware' i discovered in one of my projects that - due to other element states - my states could also be 'fail' or 'standby'...and offcourse i wanted that to be visual.

The funny thing is that - by nature - a button is a boolean...it goes 'on' or it goes 'off', and that is what a user can do.
But through either an attribute setting or through Gun/javascript the state might be changed independ of the buttons on/off status.

Try the 'ledbuttons'...the state stays 'fail' or 'standby'...The user can't change these states because they where set 'external'.

Now press the 'reset' button and try the ledbuttons again

Three state

In the above example the button can only be turned 'on/off' if the state is 'on' or 'off', but what if you need 3 states? Well then you just set the "three" attribute.

I understand that 'fail' and 'standby' are not allways appropiate but as i mentioned..these buttons are born of necessity.

The big button will go from 'off' ⇒ 'standby' ⇒ 'on' ⇒ 'standby' ⇒ 'off'
The right button is just a 'on/off' button.

If you press the big button, the smaller one will nicely follow the 3 states ( external states remember), if you press the small button - when the big one is green or off - the big one will only be turned 'on' or 'off' ( it never receives a third state)

Now set the big button in 'standby' ( yellow ) and try the smaller one...right, it doesn't work because an 'on/off' button is not allowed to change external states.
